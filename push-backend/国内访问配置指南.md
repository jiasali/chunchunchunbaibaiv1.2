# 国内免梯子访问推送服务 - 配置指南

手机打不开 `workers.dev` 时，可以通过 **自定义域名** 让推送服务在国内正常访问。

---

## 方案一：Cloudflare Worker 绑定自定义域名（推荐）

你的 push-backend 已经部署在 Cloudflare，只需绑定一个**国内可访问的域名**即可。

### 前提：你需要有一个域名

- **已有域名**：阿里云、腾讯云、Cloudflare、Namecheap 等购买的都可以
- **没有域名**：可在 腾讯云/阿里云 购买，约 30-50 元/年（如 `.cn`、`.com`）

### 步骤

#### 1. 把域名接入 Cloudflare

1. 登录 [Cloudflare](https://dash.cloudflare.com)
2. 点击 **添加站点**，输入你的域名（如 `example.com`）
3. 按提示把域名的 DNS 解析改为 Cloudflare 提供的 NS 服务器
4. 等待生效（通常几分钟到几小时）

#### 2. 为 Worker 添加自定义域名

1. 在 Cloudflare 控制台，进入 **Workers & Pages**
2. 点击你的 **tabula-proactive-push**
3. 进入 **Settings** → **Domains & Routes**
4. 点击 **Add** → **Add Custom Domain**
5. 输入子域名，例如：`push.你的域名.com`（如 `push.example.com`）
6. 保存后，Cloudflare 会自动配置 DNS

#### 3. 修改游戏里的推送地址

打开 `index.html`，找到 `PUSH_SERVER_URL`，改成你的自定义域名：

```javascript
var PUSH_SERVER_URL = 'https://push.你的域名.com';
```

例如：`var PUSH_SERVER_URL = 'https://push.example.com';`

#### 4. 重新部署游戏到 GitHub

把修改后的 `index.html` 推送到 GitHub，等 GitHub Pages 更新。

## 方案二：使用腾讯云云函数（完全国内）

如果不想买域名，可以部署到 **腾讯云云函数**，国内访问稳定，有免费额度。

### 步骤概要

1. 注册 [腾讯云](https://cloud.tencent.com) 账号
2. 开通 **云函数 SCF** 和 **对象存储 COS**
3. 使用项目中的 `push-backend-tencent` 目录（需单独创建）部署
4. 获得一个 `*.apigw.tencentcs.com` 的访问地址
5. 把该地址填入游戏的 `PUSH_SERVER_URL`

*详细步骤较多，如需此方案可联系获取完整部署文档。*

---

## 方案三：先测试自定义域名是否有效

如果你有域名但不确定能否改善访问，可以：

1. 按方案一完成 Cloudflare 自定义域名配置
2. 在手机浏览器（不开梯子）访问：`https://push.你的域名.com/vapid-public`
3. 若能打开并看到 JSON，说明配置成功，可以正常使用

## 常见问题

**Q：我没有域名怎么办？**  
A：可到 腾讯云/阿里云 购买，首年约 30 元左右。或使用方案二（腾讯云云函数）。

**Q：域名一定要备案吗？**  
A：不需要。推送服务是 API 接口，不涉及网站备案。

**Q：绑定自定义域名后，原来的 workers.dev 还能用吗？**  
A：可以。两个地址都能访问，国内用户用自定义域名即可。
