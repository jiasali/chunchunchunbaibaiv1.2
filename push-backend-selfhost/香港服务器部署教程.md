# 纯白人生 - 香港服务器自托管推送服务部署教程

把推送服务从 Cloudflare Workers 迁移到你自己的香港服务器，**国内用户无需梯子即可收到主动回复通知**。

---

## 一、为什么换成自己的香港服务器？

| 对比项 | Cloudflare Workers | 自托管香港服务器 |
|--------|-------------------|------------------|
| workers.dev 国内访问 | ❌ 经常打不开，需梯子 | ✅ 香港服务器国内直连 |
| 主动回复通知 | 需梯子才能注册/收推送 | 无需梯子 |
| 成本 | 免费 | 需购买服务器（约 30-50 元/月） |
| 维护 | 无需维护 | 需自己维护 |

**结论**：如果你有香港服务器，换成自托管后，国内用户开启「后台主动回复」时**不需要挂梯子**，注册和推送都会更稳定。

---

## 二、部署前准备

### 1. 香港服务器

- 推荐：腾讯云轻量、阿里云、Vultr 香港等
- 系统：**Ubuntu 22.04** 或 **CentOS 7+**
- 配置：1 核 1G 即可
- 需开放端口：**80**（HTTP）、**443**（HTTPS）、**3000**（或你自定义的端口，若用 Nginx 反向代理则可不对外开放 3000）

### 2. 域名（可选但推荐）

- 推送服务需要 **HTTPS**，浏览器要求安全上下文
- 可用免费证书：Let's Encrypt
- 若没有域名，可用服务器 IP + 自签名证书（部分浏览器可能不信任）

### 3. 本地已安装

- Node.js 18+（用于生成 VAPID 密钥、本地测试）
- Git（用于上传代码到服务器）

## 三、部署步骤

### 步骤 1：在服务器上安装 Node.js

```bash
# Ubuntu / Debian
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# 验证
node -v   # 应显示 v20.x
npm -v
```

### 步骤 2：上传代码到服务器

在**本地**项目目录执行：

```bash
# 方式 A：用 scp 上传整个 push-backend-selfhost 文件夹
scp -r push-backend-selfhost root@你的服务器IP:/opt/tabula-push

# 方式 B：用 Git（若项目在 GitHub）
# 在服务器上：git clone 你的仓库，然后 cd 到 push-backend-selfhost
```

### 步骤 3：在服务器上安装依赖并生成 VAPID 密钥

```bash
cd /opt/tabula-push
npm install
npm run generate-vapid
```

会输出类似：

```
VAPID_PUBLIC_KEY=BNxxxxxxxxxxxx...
VAPID_PRIVATE_KEY=xxxxxxxxxxxx...
```

**复制这两行**，后面配置环境变量要用。

### 步骤 4：配置环境变量

```bash
# 创建 .env 文件
nano .env
```

填入（把 `xxx` 换成你 generate-vapid 输出的实际值）：

```
PORT=3000
VAPID_PUBLIC_KEY=BNxxxxxxxxxxxx...
VAPID_PRIVATE_KEY=xxxxxxxxxxxx...
VAPID_SUBJECT=mailto:你的邮箱@example.com
GAME_ORIGIN=*
```

保存退出（Ctrl+O 回车，Ctrl+X）。

### 步骤 5：用 PM2 保持进程常驻

```bash
# 安装 PM2
sudo npm install -g pm2

# 启动服务（server.js 已内置 dotenv，会自动加载 .env）
pm2 start server.js --name tabula-push -i 1
pm2 save
pm2 startup   # 按提示执行，实现开机自启
```

### 步骤 6：配置 Nginx 反向代理（推荐，用于 HTTPS）

若你有域名 `push.你的域名.com`，并已解析到服务器 IP：

```bash
sudo apt install nginx certbot python3-certbot-nginx -y
sudo certbot --nginx -d push.你的域名.com
```

然后编辑 Nginx 配置：

```bash
sudo nano /etc/nginx/sites-available/tabula-push
```

写入：

```nginx
server {
    listen 443 ssl;
    server_name push.你的域名.com;
    ssl_certificate /etc/letsencrypt/live/push.你的域名.com/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/push.你的域名.com/privkey.pem;

    location / {
        proxy_pass http://127.0.0.1:3000;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

启用并重载：

```bash
sudo ln -s /etc/nginx/sites-available/tabula-push /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

### 步骤 7：修改游戏中的推送地址

编辑 `index.html`，找到 `PUSH_SERVER_URL` 和 `PUSH_SERVER_URL_CN`，改成你的地址：

```javascript
/* 主推送地址：你的香港服务器 */
var PUSH_SERVER_URL = 'https://push.你的域名.com';
/* 国内备用：若主地址在国内可访问，可留空或与上面相同 */
var PUSH_SERVER_URL_CN = '';
```

若没有域名，用 IP + 端口（需 HTTPS，可自签名证书，部分环境可能不兼容）：

```javascript
var PUSH_SERVER_URL = 'https://你的服务器IP:3000';
```

## 四、验证部署

1. 在浏览器访问：`https://push.你的域名.com/vapid-public`  
   应返回 JSON：`{"publicKey":"BNxxx..."}`

2. 访问：`https://push.你的域名.com/api/status`  
   应返回：`{"ok":true,"vapidConfigured":true,"relationCount":0}`

3. 在游戏中开启「后台主动回复」，应能正常注册，不再提示网络错误。

## 五、常见问题

**Q：没有域名怎么办？**  
A：可用服务器 IP + 自签名证书，但部分浏览器可能不信任。建议买一个便宜域名（约 30 元/年）并配置 Let's Encrypt。

**Q：部署后国内还是收不到推送？**  
A：推送链路是：你的服务器 → 浏览器厂商推送服务（如 FCM）→ 用户设备。Chrome 用的 FCM 在国内可能不稳定；Firefox 用的 Mozilla 推送服务相对好一些。但**注册**（开启后台主动回复）不再依赖梯子，这已经解决了主要痛点。

**Q：如何更新代码？**  
A：在服务器上 `cd /opt/tabula-push`，拉取新代码后 `pm2 restart tabula-push`。

**Q：数据存在哪里？**  
A：在 `push-backend-selfhost/data/` 目录，建议定期备份。

---

## 六、总结

1. 在香港服务器上安装 Node.js，上传 `push-backend-selfhost` 代码  
2. `npm install` 和 `npm run generate-vapid`，配置 `.env`  
3. 用 PM2 启动 `server.js`  
4. 用 Nginx + Let's Encrypt 配置 HTTPS  
5. 把游戏的 `PUSH_SERVER_URL` 改成你的 `https://push.你的域名.com`  

完成后，国内用户无需梯子即可使用后台主动回复功能。
